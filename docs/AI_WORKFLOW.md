# AI Workflow

## 1. Agent Loop

AI Agent 处理 UE 日志问题时遵循以下闭环：

1. 接收 UE 项目路径。
2. 调用日志分析工具读取最新日志。
3. 生成结构化报告。
4. 根据报告定位最可能的问题来源。
5. 如果需要修改代码，保持修改范围最小。
6. 修改后运行 pytest。
7. 提交前扫描最新 UE 日志，确认没有明显阻塞问题。
8. 输出变更说明、测试结果和剩余风险。

## 2. 输入

- UE 项目根目录。
- 可选：显式日志文件路径，必须在 UE 项目目录内。
- 可选：读取尾部字符数，默认 20000。

## 3. 输出

- JSON 结构化分析结果。
- Markdown 报告。
- Hook 退出码。

## 4. 分析策略

优先处理高严重度信号：

1. `Fatal error`
2. `Assertion failed`
3. `PackagingResults: Error`
4. C++ 编译器错误
5. 构建失败
6. 插件加载失败
7. Blueprint 错误
8. 资源引用错误
9. 普通 Warning

## 5. AI 修复建议原则

- 先定位第一条高严重度问题。
- 保留日志证据，不凭空猜测。
- 对 C++ 编译错误，从首个 compiler error 修起。
- 对打包问题，确认是否是 Cook、UAT、平台 SDK、插件或资源缺失。
- 对崩溃问题，优先查看调用栈和断言条件。
- 修复后重新生成日志并再次分析。
